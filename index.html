Un Guide Rapide Pour Comprendre Docker et Comment L’utiliser

Salut sur le tutoriel qui va vous aider à commencer votre développement avec Docker.

On va commencer du base sachant qu’on sais rien sur Docker. On va apprendre les fondamentaux en utilisant des exemples. A la fin on va comprendre “how to deploy a Dockerized web application stack”

*Cette tutoriel est designer pour Linux mais si vous utilisez Windows vous pouvez remplacer les commandes Linux par les équivalentes sur Windows.

Qu’est ce que est Docker?

Un peu de détails historiques:

Docker est un logiciel qui a commencé comme un projet interne de l’entreprise dotCloud. Docker a finalement grandi au point de devenir si populaire qu’il a éclipsé dotCloud. C’est pourquoi dotCloud a créé une nouvelle société appelée Docker Incorporated. Docker Inc. est une société dédiée au développement futur de Docker et de son écosystème.

Sur le site de Docker: “Package software into standardized units for development, shipment and deployment”.

Qu’est-ce que cela signifie réellement?

C’est l’essence de ce qui rend Docker utile tout personne qui possède Docker peut exécuter n’importe quel logiciel construit pour Docker.

Pourquoi utiliser Docker?

Avez-vous essayé d’installer un logiciel sur votre ordinateur et cela ne fonctionne tout simplement pas? Vous obtenez une erreur étrange, et il est impossible de le comprendre?
Peut être cette logiciel est designer a un autre OS.

Docker vient pour résoudre cette problème.

Si votre OS peut utiliser Docker donc tout application qui utilise Docker peuvent se fonctionner sur votre machine.

Qu’est-ce qu’une Docker Image?

Une image Docker est un fichier, composé de plusieurs couches, utilisé pour exécuter du code dans un conteneur Docker.

Voici un exemple de Docker Image:

docker pull ubuntu:14.04

Cette commande dit à Docker d’installer un mode d’opération comme vous êtes sur un ubuntu OS dans le container Docker.

Pour voir une liste de Docker Images utiliser la commande:

docker images

Qu’est-ce qu’une Docker Container?

Un conteneur Docker est une instance de Docker Image en cours d’exécution.

Pour lancer notre premier Container on utilise la command suivante:

docker run ubuntu:14.04 echo 'hello world'

Ceci exécute la commande dans une instance de l’environnement Ubuntu. En d’autres termes, la commande echo ‘hello world’ est exécutée dans un conteneur Ubuntu.

On va voir quelques commande utils comme:

docker ps    # Lister Docker Containers fonctionnels
docker ps -a  # Lister Docker Containers fonctionnels et ferme

Lorsqu’un processus se termine, le conteneur Docker se ferme.

Chaque fois que vous exécutez la commande docker run un nouveau Container est créé.

Pour se opérer comme on est sur un machine ubuntu et utiliser son propre Terminal on utilise la commande:

docker run -it ubuntu:14.04 /bin/bash

Maintenant on peut écrire les commandes ubuntu comme on est sur une machine ubuntu. Par exemple utiliser la fonction ls pour voire la structure des fichiers ubuntu

Il faut noter que Docker est indépendant du machine qu’il fonctionne sur.

On peut utiliser la commande:

exit

Qui command Docker à sortir du Container and on peut vérifier qu’il a arrêter par utiliser la command docker ps 

On peut utiliser la commande suivante pour initialiser le même Container

docker start CONTAINER_ID #lister dans docker ps -a

Qu’est-ce qu’un Dockerfile?

Docker vous permet de partager l’environnement dans lequel votre code est exécuté, tout comme github vous permet de partager votre code.

Un Dockerfile vous permet de documenter les étapes permettant de configurer l’environnement pour votre application. Le moteur Docker analysera votre fichier Docker et en créera une image.

Un exemple de code Dockerfile:

FROM  ubuntu:14.04   
COPY  ./test.sh  /test.sh
RUN   chmod +x /test.sh
CMD   bash /test.sh

Pour exécuter ce fichier on utilise la command suivante:

docker build . --tag test

Qu’est-ce que est “mounting in a Docker Container”?

Mounting in a Docker container est lorsque on monte un fichier du machine host dans le Container.

Le montage du système host dans un Container permet au conteneur de lire et d’écrire sur le système hôte.

Le montage vous permet de modifier des fichiers sur votre ordinateur host tout en les exécutant dans l’environnement d’un conteneur Docker.

La commande pour fair le montage est:

docker run -v <HOST_DIRECTORY>:<CONTAINER_DIRECTORY>

Qu’est-ce que est Docker Volumes?

Avec Docker Volumes vous obtenez un conteneur qui contient des données et qui est persistant. Vous pouvez attacher un volume de données à tout autre conteneur en cours d’exécution afin qu’il puisse accéder aux données.

Au lieu d’écrire sur votre système host lors du montage, vous écrivez dans un conteneur de données monté.

Qu’est-ce que est Docker-Compose?

Docker Compose coordonne les Containers à exécuter ensemble, exécutant votre logiciel en unités composables.

Ceci est connu sous le nom container orchestration

Docker-compose est écrit au format YAML, qui est un format de données similaire à JSON ou XML. YAML est conçu pour être plus lisible par l’homme. YAML est un peu comme Python en ce sens que les espaces blancs sont importants et que les éléments sont séparés par des deux points.

Qu’est-ce que est “Composability” With Microservices?

Pouvoir échanger des composants est un aspect important de la composabilité dans un système hautement découplé.

Avec les micro-services dans Docker, la mise en œuvre d’un service est totalement indépendante des autres services. Tout ce qui compte est de respecter les API contractuelles que vous avez établies.

Dockerhub est comme Github pour Docker Images

Comment déployer des conteneurs en production?

Il existe de nombreuses options pour exécuter les conteneurs en production. Vous pouvez exécuter manuellement une seule instance de votre application en installant Docker sur un seul serveur.

There are cloud options for running containers too. AWS, Azure, and Google Cloud all support running containers.

Quelles sont les meilleures pratiques pour Container?

  • Les Containers ne doivent pas contenir de données permanentes
  • Stocker des données en dehors du Container
  • Les Containers doivent faire une chose et bien le faire
  • Minimiser les couches d’image si possible lors de l’écriture de fichiers Dockerfiles
